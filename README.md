# iSpeedtest

A Flutter-based CDN speed test app with Material 3 UI, DoH-based node
resolution, selectable test IPs, and download/upload throughput measurement.

## Upload Measurement

Upload uses one Dart `HttpClient` PUT request per test connection. Single-thread
mode uses one connection, while multi-thread mode runs four connections in
parallel. During the configured upload window, the request body is generated by a
counting stream, and each chunk consumed by `HttpClient.addStream` is added to
the local uploaded-byte counter.

Displayed upload throughput is calculated locally:

```text
upload Mbps = uploaded bytes * 8 / elapsed microseconds
```

For the final sample, elapsed time is capped at the configured upload window
(8 seconds by default), so the short response wait after body upload is not part
of the displayed Mbps denominator. The response wait is only used to finish the
request and observe HTTP/network faults.

Apple CDN response timeouts or slow/5xx responses are not treated as zero-byte
uploads. Bytes already consumed from the request stream are kept as the
client-side upload sample. Server-side fields such as `Bytes`, `BPS`, or
`DurationMs` can be inspected with `tool/upload_probe.dart`, but the app does not
use them for measurement because Apple CDN does not return a reliable Mbps value
aligned with the client's upload window.

On Android, enabling a `VpnService` or VPN-like proxy can make this client-side
write counter differ from true VPN-exit egress traffic, which may cause upload
throughput to be calculated incorrectly. The client provides an optional
"绕过 VPN" setting to solve this case by binding the process to a non-VPN
network before testing; if that bind fails, the test fails explicitly instead of
silently measuring through the VPN.

## macOS Upload Probe

Run the standalone probe with the same user agent and upload-window semantics as the app:

```sh
../flutter/bin/dart tool/upload_probe.dart --ip <test-ip> --mode single --duration 8 --response-timeout 2
```
